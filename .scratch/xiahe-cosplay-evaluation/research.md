# 夏禾 cosplay 身份漂移与“美颜归因”调研

调研日期：2026-07-19

## 结论先行

原评价的**核心判断方向合理，但证据链还不足以把这张图定案，更不能把问题单独归咎于“美颜”**。

- 我没有拿到原始人物参考图、最终生成图、实际生成提示词和七轮选择记录，因此无法独立确认“眼睛变大、下颌变窄、鼻唇改变、颈部变细、磨皮过度”是否真的发生，也无法直接复核“身份未通过”。
- 如果对照图确实显示这些身份相关结构发生了实质改变，那么按 cast-me 的“身份优先”标准，判为未通过是正确的；“妆发和服装符合”不能抵消身份漂移。
- “通用美颜化”可以作为**结果表型**的描述。研究确实显示美颜会放大眼睛、改变鼻唇与下颌、平滑皮肤；但单张生成图不能证明它是由参考图美颜独自造成。
- 应拆开三件事：**参考图美颜会降低身份依据质量；提示词与生成模型会造成或放大身份漂移；最终仍被放行则是验收失败。** 三者可以同时存在，不能互相代替。
- 夏禾不存在一个可脱离版本讨论的唯一固定造型。腾讯官方游戏资料中的夏禾是长粉发、黑色短上衣、黑色短外套/长裤和黑色露指手套，并非白色无袖上衣。因此“白色无袖＋黑手套”最多是某一版本或常见 cosplay 简写，不能未经版本锁定就当成全系列通用正典。

## “是不是美颜的锅”

### 1. 美颜确实可能制造同类变化

腾讯 Beauty AR 的官方能力清单把磨皮、大眼、瘦脸/V 脸、瘦鼻、鼻翼、下巴和嘴唇调节列为标准美颜项；这说明“眼睛更大、脸鼻更精致、皮肤更平”本来就是美颜系统的典型输出，而不是中性的图像清理。[腾讯云 Beauty AR 功能说明](https://cloud.tencent.com/document/product/616/86940)

实证研究也支持这种表型：

- FAccT 2025 对 208 张人脸应用 TikTok Bold Glamour 后，女性脸平均出现眼睛轻微放大/内倾、下颌柔化、嘴唇增厚等变化；但不同性别和个体的变化并不一致。[论文 DOI](https://doi.org/10.1145/3715275.3732126)
- 一项面部测量研究发现，Bold Glamour 会缩窄鼻尖和鼻翼、增加唇部饱满度，并抹平皱纹、色沉与皮肤纹理；它同时令样本的脸宽和下颌突出度略增，说明“窄下颌”并不是所有美颜的固定结果。[PMC 全文](https://pmc.ncbi.nlm.nih.gov/articles/PMC12558173/)
- Royal Society Open Science 对 462 个身份的研究显示，美颜会同时改变眼、鼻、下巴、颧骨和嘴唇并平滑皮肤；这足以污染身份依据，却不能反推某次生成的唯一原因。[论文全文](https://pmc.ncbi.nlm.nih.gov/articles/PMC11597472/)

因此，更准确的说法是：**如果 Primary Identity Anchor 本身经过明显美颜，它已经是降质的身份证据；模型忠实继承它，也可能忠实继承错误的几何与纹理。** 但在没看到原图、也没有原始/美颜参考的对照生成前，不能说“就是美颜的锅”。

### 2. 即使参考图无美颜，生成仍会漂移

OpenAI 官方文档一方面说明 GPT Image 2 会以高保真方式处理图片输入，另一方面仍明确把“重复角色的视觉一致性”列为已知限制。高保真输入不等于生物特征级身份保证。[图片输入保真度](https://developers.openai.com/api/docs/guides/image-generation#image-input-fidelity) [已知限制](https://developers.openai.com/api/docs/guides/image-generation#limitations)

官方提示指南建议身份敏感编辑明确写出 `preserve identity/geometry`、每轮重复保留项，并采用“只改变 X，其余保持不变”的小步编辑；这恰恰表明身份漂移需要主动约束，不能假定模型会自动避免。[GPT Image 提示指南](https://developers.openai.com/cookbook/examples/multimodal/image-gen-models-prompting-guide#2-prompting-fundamentals)

学术研究也把“遵循文字条件”和“保留人物身份”之间的冲突视为身份定制生成的核心困难。[DreamIdentity, AAAI 2024](https://doi.org/10.1609/aaai.v38i2.27891)

如果实际提示词真的包含 `large eyes`、`full lips`，它们是在正向要求改变身份结构，应该首先删除。若提示词并没有这些词，就不能把它们当成已确认根因。

### 3. 放行漂移是独立的验收问题

参考图有滤镜、模型会漂移，都不能解释为什么最终结果被判为“身份通过”。若复核时已经能看到身份宽度、眼裂、鼻翼、嘴角或自然皮肤纹理的明显改变，仍然放行，就是 review gate 过宽。反过来，若没有原图与成片的并排证据，仅凭文字描述也不能证明复核必然错误。

## 夏禾造型：应先锁定版本，再谈还原度

腾讯魔方工作室群的《异人之下》官方实机演示首次展示夏禾。对约 00:45–01:10 的画面做视觉观察，可以看到长亮粉发、黑色露指手套、黑色短上衣、带暗红色拼片的黑色短外套和深色长裤；动作是俯视受伤人物、以手指托起对方下巴，表演更接近从容、戏谑和危险感。[Bilibili 官方视频](https://www.bilibili.com/video/BV1t85RzHE5w/) [TapTap 官方镜像](https://www.taptap.cn/moment/663853800930411433)

官方后续文案也直接称她为“机车皮衣，粉瞳含情，危险又迷人的全性刮骨刀”。这是对该游戏版本的第一方描述，不是全系列永恒设定。[TapTap 官方活动文案](https://www.taptap.cn/moment/664069854927523176)

由此可以可靠地说：

- 长粉发和危险、魅惑的角色读取有第一方支持；“清纯偶像写真感偏强”作为气质批评是合理方向，但仍需看实际成片。
- 黑手套在官方游戏版本中出现；白色无袖上衣并非这个版本的造型。
- 不能仅凭“粉发＋白衣＋黑手套”判断精确还原，也不能反过来说白色无袖一定错误。应先指定目标是漫画、哪一季动画、游戏，还是明确允许的 cosplay 再设计版本，再用该版本的官方帧做服装和气质基准。

## 对原评价逐项判定

| 原评价主张 | 目前判定 | 需要的证据 |
| --- | --- | --- |
| 身份复核应未通过 | **条件成立** | 原参考与成片同角度并排图；确认变化超出妆发、光线和表情范围 |
| 结果被“通用美颜化” | **表型说法有依据** | 实图确认大眼、鼻唇/下颌变化和纹理抹除 |
| 问题是参考图美颜造成 | **未证实** | 原始参考与美颜参考的对照生成；不能由单张成片反推 |
| 自然皮肤纹理方向未执行 | **若选择记录与成片如述，则成立** | 实际选择记录、提示词与高分辨率成片 |
| 夏禾气质不足 | **方向合理，未能定案** | 选定版本的官方帧与成片表情、姿态对照 |
| 白色无袖更典型 | **版本不明，表述过强** | 指定目标版本及对应第一方视觉资料 |
| 七个选择门过于表单化 | **UX 判断，和美颜归因无关** | 完整对话、每个门是否改变结果及用户成本 |
| 二手资料直接导致造型表层化 | **因果证据不足** | 实际使用的来源、brief 与生成提示；只能先说来源可靠性偏弱 |
| 增加正面与 3/4 参考必能修复 | **合理缓解，不是保证** | 同一提示条件下的重复对照实验 |

## 最小验证方法

要回答“究竟是谁的锅”，最小充分实验不是继续堆负面词，而是做一个 2×2 对照：

| | 中性身份锁提示 | 含美颜特征提示 |
| --- | --- | --- |
| 清晰、少滤镜参考 | A | B |
| 同一人物的美颜参考 | C | D |

每格使用相同构图、服装、角色版本和随机次数，重复生成数张；盲评眼裂开度、眼距、鼻翼、下颌宽度、嘴角关系、皮肤纹理和整体身份可辨识度。比较 A/C 可以估计参考美颜影响，比较 A/B 可以估计提示词影响；A 仍漂移的部分才更接近模型与生成随机性。无论根因在哪一格，明显漂移仍被通过都应另记为验收失败。

在拿到原图、成片、实际 prompt 和流程记录前，最严谨的最终结论是：**问题很可能真实存在，但当前只能确认原评价的分析框架有依据，不能确认这张具体图片的事实与责任分配；“美颜”是可能的输入污染源，不是已经证明的唯一元凶。**
